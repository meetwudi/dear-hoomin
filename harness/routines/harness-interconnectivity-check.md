# Harness Interconnectivity Check

Use this routine when adding, moving, or materially changing harness documents.

## Checklist

- Start at `harness/README.md` and confirm the changed document still fits one existing harness component.
- Check whether the durable source document points to its closest status, feature, or routine document.
- Check whether the closest status, feature, or routine document points back to the durable source document.
- Keep cross-links selective: link direct source, status, and routine neighbors; avoid linking every related document from every other document.
- Prefer one directional pointer from broad index files to active routines or feature logs, and bidirectional pointers only for documents that are expected to be used together.
- Remove stale links when a harness document is superseded, renamed, or deleted.
