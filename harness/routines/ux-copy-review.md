# UX Copy Review

Use this routine when polishing a user-facing flow or before pushing UI changes that add or rename visible copy.

## Checklist

- Scan the rendered page or browser screenshot for repeated headings, duplicate labels, redundant helper text, and visible UI terms that explain the same thing twice.
- Prefer one clear label per area. Remove decorative or section headers when the surrounding tab, page title, or form fields already explain the job.
- Check product vocabulary against `harness/product/mission-and-alignments.md`; the scheduled daily ritual is a daily musing, journals are hoomin-created entries, and the pet area is Furbaby.
- Keep helper copy only when it changes a decision or prevents a likely mistake.
- Verify the final page in the browser after removal so spacing still feels intentional.
